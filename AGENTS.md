# while: unemployed PROJECT GUIDELINES

This is a fullstack project build for course MGT-103 that plans to transform the interview experience for job seekers and recruiters alike. The platform leverages AI to streamline the interview process, making it more efficient and effective for all parties involved. It is meant to train users to speak while coding and explain their thought process clearly.

This entire project is build on the philosphy of learning by doing. The platform provides a simulated interview environment where users can practice coding interviews in real-time, receiving instant feedback and suggestions for improvement.

## Project Structure

/backend - Our backend services and APIs
/backend/agent - Our LangChain 1v agents for handling interview simulations

/frontend/tech-interview-prep
/frontend/tech-interview-prep/src/components/ - Authorization, layout and problems components
/frontend/tech-interview-prep/src/app/ - Dashboard and authorization layout, as well as main api components
/frontend/tech-interview-prep/src/lib/ - utility functions and constants as well as APIs

## Code Standards

- Follow consistent coding conventions and style guides.
- Be as concise as possible with documentation and comments
- DO NOT create MULTIPLE .md documents for changes, only provide comments in relevant code sections that were changed.
- Comments should be clear and concise, explaining the purpose and functionality of the code.
- Code itself sholud be self-explanatory wherever possible as to ease future maintenance.
